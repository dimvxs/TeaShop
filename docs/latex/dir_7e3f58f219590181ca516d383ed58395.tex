\begin{quote}
A tiny (183B to 210B) and fast utility to ascend parent directories \end{quote}


With \href{https://en.wikipedia.org/wiki/Escalade}{\texttt{escalade}}, you can scale parent directories until you\textquotesingle{}ve found what you\textquotesingle{}re looking for.~\newline
Given an input file or directory, {\ttfamily escalade} will continue executing your callback function until either\+:

1) the callback returns a truthy value 2) {\ttfamily escalade} has reached the system root directory (eg, {\ttfamily /})

\begin{quote}
{\bfseries{Important\+:}}~\newline
Please note that {\ttfamily escalade} only deals with direct ancestry – it will not dive into parents\textquotesingle{} sibling directories. \end{quote}

\hypertarget{dir_7e3f58f219590181ca516d383ed58395_autotoc_md1227}{}\doxysubsection{\texorpdfstring{API}{API}}\label{dir_7e3f58f219590181ca516d383ed58395_autotoc_md1227}
\hypertarget{dir_7e3f58f219590181ca516d383ed58395_autotoc_md1228}{}\doxysubsubsection{\texorpdfstring{escalade(input, callback)}{escalade(input, callback)}}\label{dir_7e3f58f219590181ca516d383ed58395_autotoc_md1228}
Returns\+: {\ttfamily string\texorpdfstring{$\vert$}{|}void} or {\ttfamily Promise\texorpdfstring{$<$}{<}string\texorpdfstring{$\vert$}{|}void\texorpdfstring{$>$}{>}}

When your {\ttfamily callback} locates a file, {\ttfamily escalade} will resolve/return with an absolute path.~\newline
 If your {\ttfamily callback} was never satisfied, then {\ttfamily escalade} will resolve/return with nothing (undefined).

\begin{quote}
{\bfseries{Important\+:}}~\newline
The {\ttfamily sync} and {\ttfamily async} versions share the same API.~\newline
The {\bfseries{only}} difference is that {\ttfamily sync} is not Promise-\/based. \end{quote}
\hypertarget{dir_7e3f58f219590181ca516d383ed58395_autotoc_md1229}{}\doxysubsubsubsection{\texorpdfstring{input}{input}}\label{dir_7e3f58f219590181ca516d383ed58395_autotoc_md1229}
Type\+: {\ttfamily string}

The path from which to start ascending.

This may be a file or a directory path.~\newline
However, when {\ttfamily input} is a file, {\ttfamily escalade} will begin with its parent directory.

\begin{quote}
{\bfseries{Important\+:}} Unless given an absolute path, {\ttfamily input} will be resolved from {\ttfamily process.\+cwd()} location. \end{quote}
\hypertarget{dir_7e3f58f219590181ca516d383ed58395_autotoc_md1230}{}\doxysubsubsubsection{\texorpdfstring{callback}{callback}}\label{dir_7e3f58f219590181ca516d383ed58395_autotoc_md1230}
Type\+: {\ttfamily Function}

The callback to execute for each ancestry level. It always is given two arguments\+:

1) {\ttfamily dir} -\/ an absolute path of the current parent directory 2) {\ttfamily names} -\/ a list ({\ttfamily string\mbox{[}\mbox{]}}) of contents {\itshape relative to} the {\ttfamily dir} parent

\begin{quote}
{\bfseries{Note\+:}} The {\ttfamily names} list can contain names of files {\itshape and} directories. \end{quote}


When your callback returns a {\itshape falsey} value, then {\ttfamily escalade} will continue with {\ttfamily dir}\textquotesingle{}s parent directory, re-\/invoking your callback with new argument values.

When your callback returns a string, then {\ttfamily escalade} stops iteration immediately.~\newline
 If the string is an absolute path, then it\textquotesingle{}s left as is. Otherwise, the string is resolved into an absolute path {\itshape from} the {\ttfamily dir} that housed the satisfying condition.

\begin{quote}
{\bfseries{Important\+:}} Your {\ttfamily callback} can be a {\ttfamily Promise/\+Async\+Function} when using the "{}async"{} version of {\ttfamily escalade}. \end{quote}
