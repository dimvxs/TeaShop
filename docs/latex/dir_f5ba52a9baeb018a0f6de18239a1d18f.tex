In order to analyze scope, you\textquotesingle{}ll need to have an \href{https://github.com/estree/estree}{\texttt{ESTree}} compliant AST structure to run it on. The primary method is {\ttfamily eslint\+Scope.\+analyze()}, which takes two arguments\+:


\begin{DoxyEnumerate}
\item {\ttfamily ast} -\/ the ESTree-\/compliant AST structure to analyze.
\item {\ttfamily options} (optional) -\/ Options to adjust how the scope is analyzed, including\+:
\begin{DoxyItemize}
\item {\ttfamily ignore\+Eval} (default\+: {\ttfamily false}) -\/ Set to {\ttfamily true} to ignore all {\ttfamily eval()} calls (which would normally create scopes).
\item {\ttfamily nodejs\+Scope} (default\+: {\ttfamily false}) -\/ Set to {\ttfamily true} to create a top-\/level function scope needed for Common\+JS evaluation.
\item {\ttfamily implied\+Strict} (default\+: {\ttfamily false}) -\/ Set to {\ttfamily true} to evaluate the code in strict mode even outside of modules and without {\ttfamily "{}use strict"{}}.
\item {\ttfamily ecma\+Version} (default\+: {\ttfamily 5}) -\/ The version of ECMAScript to use to evaluate the code.
\item {\ttfamily source\+Type} (default\+: {\ttfamily "{}script"{}}) -\/ The type of Java\+Script file to evaluate. Change to {\ttfamily "{}module"{}} for ECMAScript module code.
\item {\ttfamily child\+Visitor\+Keys} (default\+: {\ttfamily null}) -\/ An object with visitor key information (like \href{https://github.com/eslint/js/tree/main/packages/eslint-visitor-keys}{\texttt{{\ttfamily eslint-\/visitor-\/keys}}}). Without this, {\ttfamily eslint-\/scope} finds child nodes to visit algorithmically. Providing this option is a performance enhancement.
\item {\ttfamily fallback} (default\+: {\ttfamily "{}iteration"{}}) -\/ The strategy to use when {\ttfamily child\+Visitor\+Keys} is not specified. May be a function.
\item {\ttfamily jsx} (default\+: {\ttfamily false}) -\/ Enables the tracking of JSX components as variable references.
\end{DoxyItemize}
\end{DoxyEnumerate}

Scopes returned by the Scope\+Manager methods have the following properties\+:


\begin{DoxyItemize}
\item {\ttfamily type} -\/ The type of scope (e.\+g., "{}function"{}, "{}block"{}, "{}global"{}).
\item {\ttfamily variables} -\/ Array of variables declared in this scope.
\item {\ttfamily set} -\/ A Map of variable names to Variable objects for variables declared in this scope.
\item {\ttfamily references} -\/ Array of references in this scope.
\item {\ttfamily through} -\/ Array of references in this scope and its child scopes that aren\textquotesingle{}t resolved in this scope or its child scopes.
\item {\ttfamily variable\+Scope} -\/ Reference to the closest variable scope.
\item {\ttfamily upper} -\/ Reference to the parent scope.
\item {\ttfamily child\+Scopes} -\/ Array of child scopes.
\item {\ttfamily block} -\/ The AST node that created this scope.
\end{DoxyItemize}\hypertarget{dir_f5ba52a9baeb018a0f6de18239a1d18f_autotoc_md1275}{}\doxysubsubsubsection{\texorpdfstring{Global\+Scope}{Global\+Scope}}\label{dir_f5ba52a9baeb018a0f6de18239a1d18f_autotoc_md1275}

It extends the base {\ttfamily Scope} class with additional functionality for handling implicitly defined global variables.\hypertarget{dir_f5ba52a9baeb018a0f6de18239a1d18f_autotoc_md1276}{}\doxysubsubsubsubsection{\texorpdfstring{Properties}{Properties}}\label{dir_f5ba52a9baeb018a0f6de18239a1d18f_autotoc_md1276}
