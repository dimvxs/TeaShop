\hypertarget{dir_4903d150e4156def77f8ce8103f51f7b_autotoc_md1833}{}\doxysubsubsection{\texorpdfstring{load (string \mbox{[} , options \mbox{]})}{load (string \mbox{[} , options \mbox{]})}}\label{dir_4903d150e4156def77f8ce8103f51f7b_autotoc_md1833}
Parses {\ttfamily string} as single YAML document. Returns either a plain object, a string, a number, {\ttfamily null} or {\ttfamily undefined}, or throws {\ttfamily YAMLException} on error. By default, does not support regexps, functions and undefined.

options\+:


\begin{DoxyItemize}
\item {\ttfamily filename} {\itshape (default\+: null)} -\/ string to be used as a file path in error/warning messages.
\item {\ttfamily on\+Warning} {\itshape (default\+: null)} -\/ function to call on warning messages. Loader will call this function with an instance of {\ttfamily YAMLException} for each warning.
\item {\ttfamily schema} {\itshape (default\+: {\ttfamily DEFAULT\+\_\+\+SCHEMA})} -\/ specifies a schema to use.
\begin{DoxyItemize}
\item {\ttfamily FAILSAFE\+\_\+\+SCHEMA} -\/ only strings, arrays and plain objects\+: \href{https://www.yaml.org/spec/1.2/spec.html\#id2802346}{\texttt{https\+://www.\+yaml.\+org/spec/1.\+2/spec.\+html\#id2802346}}
\item {\ttfamily JSON\+\_\+\+SCHEMA} -\/ all JSON-\/supported types\+: \href{https://www.yaml.org/spec/1.2/spec.html\#id2803231}{\texttt{https\+://www.\+yaml.\+org/spec/1.\+2/spec.\+html\#id2803231}}
\item {\ttfamily CORE\+\_\+\+SCHEMA} -\/ same as {\ttfamily JSON\+\_\+\+SCHEMA}\+: \href{https://www.yaml.org/spec/1.2/spec.html\#id2804923}{\texttt{https\+://www.\+yaml.\+org/spec/1.\+2/spec.\+html\#id2804923}}
\item {\ttfamily DEFAULT\+\_\+\+SCHEMA} -\/ all supported YAML types.
\end{DoxyItemize}
\item {\ttfamily json} {\itshape (default\+: false)} -\/ compatibility with JSON.\+parse behaviour. If true, then duplicate keys in a mapping will override values rather than throwing an error.
\end{DoxyItemize}

NOTE\+: This function {\bfseries{does not}} understand multi-\/document sources, it throws exception on those.

NOTE\+: JS-\/\+YAML {\bfseries{does not}} support schema-\/specific tag resolution restrictions. So, the JSON schema is not as strictly defined in the YAML specification. It allows numbers in any notation, use {\ttfamily Null} and {\ttfamily NULL} as {\ttfamily null}, etc. The core schema also has no such restrictions. It allows binary notation for integers.\hypertarget{dir_4903d150e4156def77f8ce8103f51f7b_autotoc_md1834}{}\doxysubsubsection{\texorpdfstring{load\+All (string \mbox{[}, iterator\mbox{]} \mbox{[}, options \mbox{]})}{load\+All (string \mbox{[}, iterator\mbox{]} \mbox{[}, options \mbox{]})}}\label{dir_4903d150e4156def77f8ce8103f51f7b_autotoc_md1834}

\hypertarget{dir_4903d150e4156def77f8ce8103f51f7b_autotoc_md1835}{}\doxysubsubsection{\texorpdfstring{dump (object \mbox{[} , options \mbox{]})}{dump (object \mbox{[} , options \mbox{]})}}\label{dir_4903d150e4156def77f8ce8103f51f7b_autotoc_md1835}
Serializes {\ttfamily object} as a YAML document. Uses {\ttfamily DEFAULT\+\_\+\+SCHEMA}, so it will throw an exception if you try to dump regexps or functions. However, you can disable exceptions by setting the {\ttfamily skip\+Invalid} option to {\ttfamily true}.

options\+:


\begin{DoxyItemize}
\item {\ttfamily indent} {\itshape (default\+: 2)} -\/ indentation width to use (in spaces).
\item {\ttfamily no\+Array\+Indent} {\itshape (default\+: false)} -\/ when true, will not add an indentation level to array elements
\item {\ttfamily skip\+Invalid} {\itshape (default\+: false)} -\/ do not throw on invalid types (like function in the safe schema) and skip pairs and single values with such types.
\item {\ttfamily flow\+Level} {\itshape (default\+: -\/1)} -\/ specifies level of nesting, when to switch from block to flow style for collections. -\/1 means block style everwhere
\item {\ttfamily styles} -\/ "{}tag"{} =\texorpdfstring{$>$}{>} "{}style"{} map. Each tag may have own set of styles.
\item {\ttfamily schema} {\itshape (default\+: {\ttfamily DEFAULT\+\_\+\+SCHEMA})} specifies a schema to use.
\item {\ttfamily sort\+Keys} {\itshape (default\+: {\ttfamily false})} -\/ if {\ttfamily true}, sort keys when dumping YAML. If a function, use the function to sort the keys.
\item {\ttfamily line\+Width} {\itshape (default\+: {\ttfamily 80})} -\/ set max line width. Set {\ttfamily -\/1} for unlimited width.
\item {\ttfamily no\+Refs} {\itshape (default\+: {\ttfamily false})} -\/ if {\ttfamily true}, don\textquotesingle{}t convert duplicate objects into references
\item {\ttfamily no\+Compat\+Mode} {\itshape (default\+: {\ttfamily false})} -\/ if {\ttfamily true} don\textquotesingle{}t try to be compatible with older yaml versions. Currently\+: don\textquotesingle{}t quote "{}yes"{}, "{}no"{} and so on, as required for YAML 1.\+1
\item {\ttfamily condense\+Flow} {\itshape (default\+: {\ttfamily false})} -\/ if {\ttfamily true} flow sequences will be condensed, omitting the space between {\ttfamily a, b}. Eg. {\ttfamily \textquotesingle{}\mbox{[}a,b\mbox{]}\textquotesingle{}}, and omitting the space between {\ttfamily key\+: value} and quoting the key. Eg. {\ttfamily \textquotesingle{}\{"{}a"{}\+:b\}\textquotesingle{}} Can be useful when using yaml for pretty URL query params as spaces are \%-\/encoded.
\item {\ttfamily quoting\+Type} {\itshape ({\ttfamily \textquotesingle{}} or {\ttfamily "{}}, default\+: {\ttfamily \textquotesingle{}})} -\/ strings will be quoted using this quoting style. If you specify single quotes, double quotes will still be used for non-\/printable characters.
\item {\ttfamily force\+Quotes} {\itshape (default\+: {\ttfamily false})} -\/ if {\ttfamily true}, all non-\/key strings will be quoted even if they normally don\textquotesingle{}t need to.
\item {\ttfamily replacer} -\/ callback {\ttfamily function (key, value)} called recursively on each key/value in source object (see {\ttfamily replacer} docs for {\ttfamily JSON.\+stringify}).
\end{DoxyItemize}
