"{}\+Fast Refresh"{}, also known as "{}hot reloading"{}, is a feature in many modern bundlers. If you update some React component(s) on disk, then the bundler will know to update only the impacted parts of your page -- without a full page reload.

{\ttfamily eslint-\/plugin-\/react-\/refresh} enforces that your components are structured in a way that integrations such as \href{https://www.npmjs.com/package/react-refresh}{\texttt{react-\/refresh}} expect.\hypertarget{dir_ee82ef395ef160bc60d1e2944114cbf3_autotoc_md1251}{}\doxysubsubsection{\texorpdfstring{Limitations}{Limitations}}\label{dir_ee82ef395ef160bc60d1e2944114cbf3_autotoc_md1251}
⚠️ To avoid false positives, by default this plugin is only applied on {\ttfamily tsx} \& {\ttfamily jsx} files. See Options to run on JS files. ⚠️

The plugin relies on naming conventions (i.\+e. use Pascal\+Case for components, camel\+Case for util functions). This is why there are some limitations\+:

\item {\ttfamily export \texorpdfstring{$\ast$}{*}} are not supported and will be reported as an error
