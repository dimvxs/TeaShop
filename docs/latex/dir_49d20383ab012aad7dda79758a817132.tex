\hypertarget{dir_49d20383ab012aad7dda79758a817132_autotoc_md960}{}\doxysubsubsection{\texorpdfstring{Adding SVGs}{Adding SVGs}}\label{dir_49d20383ab012aad7dda79758a817132_autotoc_md960}
Icons are typically only added by @mdo, but exceptions can be made. New glyphs are designed in Figma first on a 16x16px grid, then exported as flattened SVGs with {\ttfamily fill} (no stroke). Once a new SVG icon has been added to the {\ttfamily icons} directory, we use an npm script to\+:


\begin{DoxyEnumerate}
\item Optimize our SVGs with SVGO.
\item Modify the SVGs source code, removing all attributes before setting new attributes and values in our preferred order.
\end{DoxyEnumerate}

Use {\ttfamily npm run icons} to run the script, run {\ttfamily npm run pages} to build permalink pages, complete those pages, and, finally, commit the results in a new branch for updating.

{\bfseries{Warning}}\+: Please exclude any auto-\/generated files, like {\ttfamily font/\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}} and {\ttfamily bootstrap-\/icons.\+svg} from your branch because they cause conflicts, and we generally update the dist files before a release.\hypertarget{dir_49d20383ab012aad7dda79758a817132_autotoc_md961}{}\doxysubsubsection{\texorpdfstring{Publishing}{Publishing}}\label{dir_49d20383ab012aad7dda79758a817132_autotoc_md961}
