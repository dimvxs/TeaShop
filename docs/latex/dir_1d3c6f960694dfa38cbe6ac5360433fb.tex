\hypertarget{dir_1d3c6f960694dfa38cbe6ac5360433fb_autotoc_md2608}{}\doxysubsection{\texorpdfstring{reusify}{reusify}}\label{dir_1d3c6f960694dfa38cbe6ac5360433fb_autotoc_md2608}
\href{https://badge.fury.io/js/reusify}{\texttt{}}

Reuse your objects and functions for maximum speed. This technique will make any function run \texorpdfstring{$\sim$}{\string~}10\% faster. You call your functions a lot, and it adds up quickly in hot code paths.

Also note how in the above examples, the code, that consumes an instance of {\ttfamily My\+Object}, reset the state to initial condition, just before storing it in the cache. That\textquotesingle{}s needed so that every subsequent request for an instance from the cache, could get a clean instance.\hypertarget{dir_1d3c6f960694dfa38cbe6ac5360433fb_autotoc_md2610}{}\doxysubsection{\texorpdfstring{Why}{Why}}\label{dir_1d3c6f960694dfa38cbe6ac5360433fb_autotoc_md2610}
It is faster because V8 doesn\textquotesingle{}t have to collect all the functions you create. On a short-\/lived benchmark, it is as fast as creating the nested function, but on a longer time frame it creates less pressure on the garbage collector.\hypertarget{dir_1d3c6f960694dfa38cbe6ac5360433fb_autotoc_md2611}{}\doxysubsection{\texorpdfstring{Other examples}{Other examples}}\label{dir_1d3c6f960694dfa38cbe6ac5360433fb_autotoc_md2611}
